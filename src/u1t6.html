1.6 Generación de la tabla de símbolo y tabla de direcciones

La generación de la tabla de símbolos y la tabla de direcciones son dos componentes fundamentales en el proceso de compilación de un programa.

Tabla de Símbolos: Una tabla de símbolos es una estructura de datos que almacena toda la información de los identificadores del lenguaje fuente. Los atributos de un identificador pueden obtenerse directamente del análisis del programa fuente o de forma implícita a través del contexto en el que aparece el elemento en el programa fuente. Los atributos pueden incluir el nombre del elemento, el tipo del elemento, la dirección de memoria en que se almacenará su valor en tiempo de ejecución, el valor del elemento y el número de dimensiones (si el elemento es un array).

Tabla de Direcciones: La tabla de direcciones es una estructura de datos que se utiliza en los switches para mantener un registro de las direcciones MAC de los dispositivos conectados a la red. La dirección MAC es un identificador único asignado a cada tarjeta de red en un dispositivo y se utiliza para identificar y comunicarse con dispositivos en la red local (LAN).

Un ejemplo simplificado de cómo podría verse una tabla de símbolos para un fragmento de código:

Nombre Tipo Ámbito Dirección
a int global 0x001
b int global 0x005
suma func global 0x009
c int suma 0x00D
d int suma 0x011

En este ejemplo, a y b son variables enteras en el ámbito global, mientras que c y d son variables enteras dentro de la función suma. Las direcciones son solo ilustrativas y representan la ubicación en memoria de cada identificador.

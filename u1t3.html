Comprobaciones de tipos en expresiones

Las comprobaciones de tipos en expresiones son esenciales en programación para garantizar que los datos manipulados sean del tipo correcto, evitando errores como la concatenación de tipos incompatibles o operaciones matemáticas inválidas. Estas comprobaciones pueden ser de tres tipos:


  1. Comprobación estática: Se realiza durante la fase de compilación y se basa en el uso de artefactos y mecanismos propios de dicha fase.
  2. Comprobación dinámica: No se realiza durante la fase de compilación y se delega al momento de la ejecución del programa.
  3. Comprobación mixta: Combina aspectos de las comprobaciones estáticas y dinámicas.

Además, existen dos tareas importantes relacionadas con las comprobaciones de tipos:


  • Inferencia de tipos: En lenguajes sin tipificación de variables o con sobrecarga, se aplican tareas de inferencia de tipos en el nivel gramatical de las expresiones para resolver el tipo de datos de la expresión resultante en función del contexto de evaluación.
  • Verificación de tipos: Comprueba la compatibilidad de tipos de todas las expresiones del código fuente recuperando la información durante la gestión de declaraciones.

Un ejemplo de comprobación de tipos en expresiones:

int c, d;


int f (int a, int b) {

if (c > 0)

return (a+b)*d*c;

}

En este código, si intentamos compilarlo y c no es de tipo int, obtendremos un error de tipo en la línea 4: Type Error at line 4: Incompatible types with operator '>'
